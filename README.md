# Canvas SDLC Inicial

## Ideia do Software

O projeto escolhido é um **Sistema de Agenda do LEDS**

A ideia é criar uma aplicação simples para cadastrar e visualizar atividades do laboratório, como reuniões, apresentações, eventos e prazos importantes

## 1. Problema a resolver

As atividades do LEDS podem ficar espalhadas em mensagens, anotações ou planilhas. Isso dificulta a organização e pode fazer com que alunos e professores esqueçam compromissos importantes

O sistema busca centralizar essas informações em uma agenda simples

## 2. Usuários principais

- Alunos do LEDS
- Professores do LEDS

## 3. Requisitos funcionais iniciais

- Cadastrar uma atividade na agenda;
- Listar as atividades cadastradas;
- Visualizar as informações principais da atividade;
- Remover uma atividade cadastrada.

## 4. Requisitos não funcionais iniciais

- Ser simples de usar;
- Ter uma interface clara;
- Funcionar em navegador web;
- Organizar as informações de forma fácil de entender;
- Ter um código simples para manutenção.

## 5. Modelo de SDLC escolhido

O modelo de SDLC escolhido foi o **modelo ágil**.

## 6. Justificativa da escolha do modelo

O modelo ágil foi escolhido porque permite adaptar o sistema conforme as necessidades reais dos alunos e professores do LEDS forem identificadas durante o uso.
Primeiro será criada uma versão simples da agenda. Depois, caso necessário, novas funcionalidades poderão ser adicionadas com base no feedback dos usuários.

## 7. Primeira entrega ou primeiro incremento planejado

A primeira entrega será uma versão básica do sistema contendo:

- Cadastro de atividades;
- Listagem das atividades;
- Visualização das informações da atividade;
- Remoção de atividades.

Essa primeira versão terá como objetivo testar se a ideia da agenda é útil para organizar as atividades do LEDS.

## 8. Riscos iniciais do projeto

Alguns riscos iniciais do projeto são:

- Falta de padronização no preenchimento das atividades;
- Conflitos de horário entre atividades cadastradas;
- Dificuldade dos usuários em entender a organização da agenda;

## 9. Estratégia de validação com o usuário

A validação será feita com alunos e professores do LEDS.
Eles poderão testar a agenda cadastrando e visualizando atividades. Depois, será coletado um feedback simples para verificar se o sistema é fácil de usar e se ajuda na organização das atividades.
